Aula 10 - Software de Versionamento

Software de Versionamento

Sistema que tem por objetivo realizar o controle de versões de um código ou documento.
É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedore.

Software de Controle de versões

Quando desenvolvemos um projeto, é possível desenvolve-lo numa pasta em nosso computador e quando vamos atualiza-lo geramos uma nova versão desse arquivo para executar as alterações. Esse processo pode se repetir por inúmeras vezes enquanto estamos gerando novas versões e fazendo novas mudanças, sendo criado vários arquivos. Essa maneira dificulta a visualização e controle das versões. Para tornar esse processo mais eficiente existe o Software de Controle de versões.

O software cria um repositório e nele há o histórico de versões de determinado arquivo, versionando o projeto. O repositório pode ser tanto local como remoto, que serão sincronizados, facilitando trabalhos realizados em equipe.

Tipos de Software de Controle de Versão

Centralizado
Possui um repositório central em um servidor com controle de versões. Dependente do servidor.
Software Subversion
Distribuído
Possui um repositório local com controle de versões. Independente do servidor.
Software Mercurial, Git

Git

É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux, foi projetado e desenvolvido por Linus Torvalds em 2005.
Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central.

GitHub

  • Plataforma de hospedagem de código-fonte e arquivos com controle de versão.
  • Surgiu em 2008 e foi adquirido em 2018 pela Microsoft por US$ 7,5 bilhões.
  • 100 milhões de projetos
  • 56 milhões de usuários

Comandos Git

commit
Torna permanente um conjunto de alterações em um repositório
push
Envia alterações para o servidor no repositório remoto
fetch
verifica se o seu repositório local está em sincronia com o remoto
pull
sicroniza o repositório local com o remoto atualizando as versões
